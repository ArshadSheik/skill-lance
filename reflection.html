"Show a browser warning when the user leaves mid-quiz, but only after they start answering. Remove the warning after submission."

#03 Algorithms

Quiz Question Shuffling — Biased Sort Bug

"Randomise my quiz questions array each time the quiz loads. Show the JavaScript."

AI Error: Suggested a sort-based shuffle — syntactically valid but statistically biased. sort()'s comparison contract is violated by a random comparator.

AI (biased) → My fix: Fisher-Yates (main.js) 
// ❌ AI suggestion — biased shuffle
[...arr].sort(() => Math.random() - 0.5)

// ✅ Fisher-Yates — uniform permutation
function shuffle(arr) {
  const a = [...arr];
  for (let i = a.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [a[i], a[j]] = [a[j], a[i]];
  }
  return a;
}

My Fix: Replaced with Fisher-Yates after noticing question clustering in testing and independently researching unbiased shuffle algorithms on MDN.

❌ AI flawed🔬 Researched✅ Fisher-Yates used

#11 CSS Animations

Confetti Celebration — DOM Memory Leak

"Trigger a confetti rain on quiz pass using only CSS keyframe animations and JS-created DOM elements. No third-party libraries."

AI Output: Generated launchConfetti() spawning DOM elements with randomised --drift and --start-rot CSS custom properties per element. Technically clever, but never removed any elements from the DOM.

Per-piece self-removal fix (main.js) 
// ❌ AI — pieces spawned but never removed from DOM

// ✅ My fix — each piece computes its own lifetime
//    and removes itself once its animation ends
const lifetime =
  (parseFloat(duration) + parseFloat(delay) + 0.5) * 1000;
setTimeout(() => piece.remove(), lifetime);

Performance Issue: AI's version left all confetti elements permanently in the DOM — accumulating on every retake. I added a per-piece setTimeout that calculates each piece's exact lifetime from its own animationDuration and animationDelay, removing it the instant its animation ends — verifiable in main.js.

#12 SVG & Canvas Animation

Quiz Score Ring — SVG Stroke Timing Bug

"Display the quiz result as an animated circular arc that fills to the percentage scored using only SVG and CSS. No chart libraries."

AI Output: Correctly calculated stroke-dasharray = 2πr and the stroke-dashoffset formula to represent any percentage. SVG markup and CSS transition were both structurally correct.

requestAnimationFrame fix (main.js) 
// ❌ AI — offset set before element was painted
ring.style.strokeDashoffset = offset; // transition never fires

// ✅ My fix — defer until after browser paint cycle
requestAnimationFrame(() => {
  const ring = resultContent.querySelector('.score-ring-fill');
  if (ring) ring.style.strokeDashoffset = offset;
});

Bug Found: Setting strokeDashoffset synchronously immediately after innerHTML assignment meant the browser had not yet painted the SVG element — the CSS transition never triggered. Wrapping in requestAnimationFrame defers the assignment until post-paint, allowing the animation to play.

✅ SVG math correct🐛 Paint timing bug✏️ rAF wrapper

Critical Findings

Identifying AI Errors & Limitations

🐛 Silent Algorithmic Bias
The sort-based shuffle is syntactically valid but mathematically biased — it violates the comparison contract of Array.sort(). Only caught through testing and independent MDN research.
⚠️ Context Blindness
When generating CSS, tools consistently produced fresh variable names instead of using existing design tokens. Every CSS block required manual remapping to the :root token system.
📂 Path & Environment Errors
AI tools have no knowledge of project file structure. Wrong relative paths and missing environment context were consistent blind spots requiring browser verification.
♿ Accessibility Gaps
Generated components consistently lacked ARIA attributes, focus management, and keyboard support. Every interactive element needed a manual accessibility audit after AI generation.
🔁 State Management Timing
Event listeners were attached unconditionally at page load. The beforeunload listener and confetti spawning both needed explicit timing guards added manually.
💾 Performance & DOM Leaks
AI-generated animations never cleaned up DOM elements on repeat runs. Confetti pieces accumulated across quiz retakes. Performance concerns like DOM size and animation cost were consistently overlooked.
🎨 Inline Style Proliferation
AI frequently generated excessive inline style="" attributes instead of adding CSS classes. This made theming, dark/light mode overrides, and maintenance significantly harder across the project.
📡 API Response Assumptions
AI assumed API responses always matched the documented shape — no checks for missing fields, unexpected types, or empty arrays. Every external fetch needed defensive validation added manually.
⏱️ Paint Timing Blindness
The SVG score ring offset was set synchronously after innerHTML assignment, before the browser had painted the element. CSS transitions never fired — only caught during live browser testing.
🧼 Hardcoded Magic Values
AI regularly used literal pixel values, z-index integers, and colour hex codes instead of referencing the :root CSS custom property system — requiring a manual pass to replace every instance.

Analysis

Reliability Assessment

Across 12 interactions, AI reliability varied significantly by task type. Standard web APIs (fetch, localStorage, IntersectionObserver) were handled accurately in most cases. Project-specific integration — file paths, existing CSS tokens, and performance constraints — consistently required correction.

The critical risk is confident incorrectness: AI output is always authoritative in tone, even when factually wrong. Every piece of generated code was tested in a real Chromium browser before being kept in the project.
