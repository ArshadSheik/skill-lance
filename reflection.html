#03 Algorithms · ChatGPT

Quiz Question Shuffling — Biased Sort Bug

"Randomise my quiz questions array each time the quiz loads. Show the JavaScript."

AI Error: Suggested a sort-based shuffle — syntactically valid but statistically biased. sort()'s comparison contract is violated by a random comparator.

AI (biased) → My fix: Fisher-Yates (main.js) 
// ❌ AI suggestion — biased shuffle
[...arr].sort(() => Math.random() - 0.5)

// ✅ Fisher-Yates — uniform permutation
function shuffle(arr) {
  const a = [...arr];
  for (let i = a.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [a[i], a[j]] = [a[j], a[i]];
  }
  return a;
}

My Fix: Replaced with Fisher-Yates after noticing question clustering in testing and independently researching unbiased shuffle algorithms on MDN.

❌ AI flawed🔬 Researched✅ Fisher-Yates used

Critical Findings

Identifying AI Errors & Limitations

🐛 Silent Algorithmic Bias
The sort-based shuffle is syntactically valid but mathematically biased — it violates the comparison contract of Array.sort(). Only caught through testing and independent MDN research.
⚠️ Context Blindness
When generating CSS, tools consistently produced fresh variable names instead of using existing design tokens. Every CSS block required manual remapping to the :root token system.
📂 Path & Environment Errors
AI tools have no knowledge of project file structure. Wrong relative paths and missing environment context were consistent blind spots requiring browser verification.
♿ Accessibility Gaps
Generated components consistently lacked ARIA attributes, focus management, and keyboard support. Every interactive element needed a manual accessibility audit after AI generation.
🔁 State Management Timing
Event listeners were attached unconditionally at page load. The beforeunload listener and confetti spawning both needed explicit timing guards added manually.
💾 Performance & DOM Leaks
AI-generated animations never cleaned up DOM elements on repeat runs. Confetti pieces accumulated across quiz retakes. Performance concerns like DOM size and animation cost were consistently overlooked.

Analysis

Reliability Assessment

Across 11 interactions, AI reliability varied significantly by task type. Standard web APIs (fetch, localStorage, IntersectionObserver) were handled accurately in most cases. Project-specific integration — file paths, existing CSS tokens, and performance constraints — consistently required correction.

The critical risk is confident incorrectness: AI output is always authoritative in tone, even when factually wrong. Every piece of generated code was tested in a real Chromium browser before being kept in the project.
