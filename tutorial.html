HTML — Structure

HTML (HyperText Markup Language) is the foundation of every web page. It uses a system of elements — made up of opening and closing tags — to give content meaning and structure. Every website you've ever visited began as HTML.

HTML tells the browser what the content is: a heading, a paragraph, a link, an image. CSS then decides how it looks, and JavaScript makes it behave.

Document Structure

Every valid HTML document follows a required structure. Missing any of these pieces can cause browsers to render your page incorrectly.

index.html 
<!-- DOCTYPE tells the browser this is HTML5 -->
<!DOCTYPE html>

<html lang="en">          <!-- lang attr aids screen readers & SEO -->
  <head>             <!-- Metadata: not shown on page -->
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>My First Page</title>    <!-- Appears in browser tab -->
    <link rel="stylesheet" href="style.css" />
  </head>
  <body>             <!-- Everything visible goes here -->
    <h1>Hello, World!</h1>
    <p>My first web page.</p>
    <script defer src="script.js"></script>
  </body>
</html>
💡 Key rule: Always include lang on <html> and a viewport meta tag. These two lines improve accessibility and ensure your site works correctly on mobile devices.

The DOCTYPE Declaration

The very first line of every HTML file must be the DOCTYPE declaration. It is not an HTML tag — it is an instruction to the browser telling it which version of HTML the page uses.

doctype.html 
<!-- Always the very first line — before anything else -->
<!DOCTYPE html>

<!-- Without this, browsers enter "quirks mode" and render inconsistently -->
What it does
Tells the browser: "render this page using the modern HTML5 standard." Without it, the browser falls back to quirks mode — an old compatibility mode that mimics buggy 1990s behaviour.
Is it an HTML tag?
No. It starts with <! not <, so it is a declaration, not an element. It has no closing tag and no attributes.
HTML5 vs older versions
Older HTML versions (HTML 4, XHTML) required long complex DOCTYPE strings. HTML5 simplified it to just <!DOCTYPE html> — short and easy to remember.

Tags & Attributes

HTML is written using tags. Most elements have an opening tag and a closing tag with content in between. Some elements are void elements — they have no content and no closing tag.

tags.html 
<!-- Opening + closing tag (most elements work this way) -->
<p>This is a paragraph.</p>

<!-- Attributes go inside the opening tag: name="value" -->
<a href="https://example.com" target="_blank">Visit Example</a>

<!-- Void elements — self-closing, no content, no closing tag -->
<img src="photo.jpg" alt="A mountain landscape" />
<br />          <!-- Line break -->
<hr />          <!-- Horizontal rule (divider line) -->
<input type="text" placeholder="Type here…" />

<!-- Boolean attributes — presence = true, no value needed -->
<input type="checkbox" checked />
<button disabled>Can't click me</button>
💡 Nesting rule: Tags must be properly nested — always close the innermost tag first. <p><strong>correct</strong></p> ✅   <p><strong>wrong</p></strong> ❌

The <head> Element

The <head> contains metadata — information about the page that the browser needs but does not display on screen. Think of it as the page's configuration block.

head.html 
<head>
  <!-- Character encoding — always UTF-8 -->
  <meta charset="UTF-8" />

  <!-- Responsive viewport — essential for mobile -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />

  <!-- Page title shown in browser tab and search results -->
  <title>My Page Title</title>

  <!-- SEO description shown in Google search results -->
  <meta name="description" content="A short summary of the page (under 160 chars)." />

  <!-- Link to an external CSS stylesheet -->
  <link rel="stylesheet" href="style.css" />

  <!-- Favicon (the small icon in the browser tab) -->
  <link rel="icon" href="favicon.ico" />
</head>
<meta charset>
Tells the browser to use UTF-8 encoding — supports every character, symbol, and emoji from all languages. Always the first tag inside <head>.
<meta viewport>
Without this, mobile browsers zoom out to show a desktop-sized view. This tag tells them to match the device's screen width and start at 1× zoom.
<title>
The text shown in the browser tab, bookmarks, and Google search results. Keep it descriptive and under 60 characters.
<link rel="stylesheet">
Loads an external CSS file. The browser fetches and applies it before rendering the page, so styling is ready immediately.

The <body> Element

Everything visible on the page lives inside the <body>. Text, images, buttons, videos, forms — if the user can see or interact with it, it belongs in the body. JavaScript <script> tags also go here, placed at the bottom so the visible content loads first.

body.html 
<body>

  <!-- Page header with navigation -->
  <header>
    <nav>
      <a href="/">Home</a>
      <a href="/about">About</a>
    </nav>
  </header>

  <!-- Main visible content -->
  <main>
    <h1>Welcome to My Site</h1>
    <p>This text is visible to the user.</p>
    <img src="hero.jpg" alt="Hero banner" />
  </main>

  <!-- Footer -->
  <footer>
    <p>&copy; 2026 My Website</p>
  </footer>

  <!-- Scripts go last — page content loads before JavaScript runs -->
  <script defer src="script.js"></script>

</body>
✅ Why scripts go at the bottom: If you put <script> in the <head> without defer, the browser pauses rendering while it downloads and runs your JS. Placing scripts before </body> (or using defer) ensures the HTML is fully parsed and visible first, giving users a faster perceived load time.

Common HTML Elements

HTML has over 100 elements. These are the most important ones you'll use every day:

<h1> → <h6>
Headings ranked by importance. Use only one <h1> per page.
<p>
Paragraph of text. Block-level — browsers add spacing above and below.
<a href="">
Hyperlink. The href attribute sets the destination URL.
<img src="" alt="">
Embeds an image. Always write descriptive alt text for accessibility.
<ul> <ol>
Unordered (bullets) and ordered (numbered) lists. Children must be <li>.
<button>
A clickable button. Always prefer <button> over <div onclick>.
<input>
Form input field. Use type attribute: text, email, checkbox, range, etc.
<div> <span>
Generic containers with no semantic meaning. Used for grouping and styling.
<form>
Groups user input fields such as text boxes, buttons, checkboxes, and dropdowns for form submissions.

Semantic HTML5

HTML5 introduced semantic elements — tags that describe the purpose of their content, not just its appearance. Search engines, screen readers, and other tools use these to understand your page.

  • <header>Introductory content, site logo, or navigation. Appears at the top of a page or section.
  • <nav>Primary navigation links. Screen readers use this to help users skip to navigation.
  • <main>The primary content of the document. Only one <main> per page.
  • <section>A thematic grouping of content, typically with a heading. More meaningful than <div>.
  • <article>Self-contained content that makes sense standalone — a blog post, news story, or comment.
  • <aside>Content tangentially related to the main content — sidebars, pull quotes, or related links.
  • <footer>Footer information: copyright, authorship, site links. Can appear on a page or within a section.
✅ Best practice: Use semantic elements instead of generic <div> wherever possible. A page built with <header>, <main>, <nav>, and <footer> is more accessible and SEO-friendly than one built entirely with <div> tags.

Forms & User Input

Forms collect user input and send it to a server. They're essential for login pages, search bars, contact forms, surveys, and any interactive data collection. Every <input> should have a <label> for accessibility.

forms.html 
<form action="/submit" method="POST">
  <!-- Text input with associated label -->
  <label for="username">Username:</label>
  <input type="text" id="username" name="username" required />

  <!-- Password input (dots hide text) -->
  <label for="password">Password:</label>
  <input type="password" id="password" name="password" />

  <!-- Email input (browser validates format) -->
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" placeholder="you@example.com" />

  <!-- Dropdown select -->
  <label for="country">Country:</label>
  <select id="country" name="country">
    <option value="">Select...</option>
    <option value="au">Australia</option>
    <option value="uk">United Kingdom</option>
  </select>

  <!-- Submit button -->
  <button type="submit">Sign Up</button>
</form>

CSS — Presentation

CSS (Cascading Style Sheets) controls how your HTML looks — colours, fonts, sizes, spacing, layout, and animations. Without CSS, the web would be plain text.

The word cascading refers to how multiple rules can apply to one element, with a specific order of priority (specificity) determining which wins.

How CSS Works

CSS is written as rules. Each rule targets one or more HTML elements and declares what styles to apply. There are three ways to add CSS to a page:

adding-css.html 
<!-- 1. External stylesheet (recommended) — keeps CSS in a separate file -->
<link rel="stylesheet" href="style.css" />

<!-- 2. Internal <style> block — CSS lives inside the HTML file -->
<style>
  h1 { color: red; }
</style>

<!-- 3. Inline style — applied directly to one element (avoid for large projects) -->
<p style="color: blue; font-size: 18px;">Inline styled text</p>
External (recommended)
A .css file linked with <link>. One file can style the whole site — change one rule and it updates every page.
Internal <style>
CSS written inside a <style> tag in the <head>. Useful for single-page projects or quick prototypes.
Inline style
Written directly on the element with the style attribute. Highest specificity — overrides everything. Hard to maintain at scale.
The Cascade
When multiple rules target the same element, CSS uses specificity and source order to decide which wins. Inline > ID > class > element.
Selectors
Used to target HTML elements for styling. Common selectors include element, class (.class), ID (#id), and descendant selectors (div p).
Box Model
Every element is a box made of content, padding, border, and margin. Understanding spacing is essential for layout design.

CSS Syntax

Every CSS rule follows the same pattern: a selector picks which elements to style, and a declaration block contains one or more property: value; pairs.

syntax.css 
/*  selector  {  property: value;  }  */

h1 {
  color:       #818cf8;    /* hex colour */
  font-size:   2rem;       /* relative unit */
  font-weight: 700;        /* bold */
  margin:      0 0 1rem;   /* top right bottom left shorthand */
}

/* Multiple selectors — same rule applies to both */
h1, h2 {
  font-family: 'Syne', sans-serif;
}

/* Comments in CSS use slash-star notation */

Colours & Units

CSS supports many ways to express colours and measurements. Choosing the right unit matters — some scale with the user's browser settings, others don't.

Hex colours
#818cf8 — six hex digits encoding red, green, blue. Short form: #fff = #ffffff. Most common in web design.
RGB & RGBA
rgb(129, 140, 248) — decimal red, green, blue values 0–255. Add an alpha channel for transparency: rgba(129, 140, 248, 0.5).
px (pixels)
Fixed size — 16px is always 16 pixels regardless of screen or settings. Good for borders and shadows; avoid for font sizes.
rem
Relative to the root font size (usually 16px). 1rem = 16px. Use for font sizes and spacing — respects user browser settings.
em
Relative to the parent element's font size. Can compound unexpectedly when nested. Use rem for most cases instead.
% (percent)
Relative to the parent element's size. width: 50% makes an element half the width of its container. Essential for responsive layouts.
💡 Best practice: Use rem for font sizes and spacing, % or fr for layout widths, and px only for fine details like borders. This makes your site accessible and easier to scale.

Selectors & Rules

A CSS rule has two parts: a selector (which elements to target) and a declaration block (the styles to apply).

p { }
Targets all <p> elements. Lowest specificity — easy to override.
.card { }
Targets all elements with class="card". Can apply to many elements.
#hero { }
Targets the single element with id="hero". High specificity.
nav a { }
Descendant — all <a> links inside any <nav> element.
a:hover { }
Pseudo-class — applies only when the user hovers their mouse over the link.

The CSS Box Model

Every HTML element is rendered as a rectangular box. The box model describes four layers that surround every element. Understanding this is the single most important concept in CSS layout.

  • contentThe actual text, image, or other content inside the element. Width and height set the size of this layer.
  • paddingTransparent space inside the border. Adds breathing room between content and border. Has the element's background colour.
  • borderA visible (or invisible) line around the padding. Can be styled with width, colour, and style.
  • marginTransparent space outside the border. Creates distance between adjacent elements. Always transparent.
⚠️ Common pitfall: By default, width only sets the content width. Add box-sizing: border-box to your CSS reset so that padding and border are included in the element's declared width.

Flexbox & Grid Layouts

Flexbox is a one-dimensional layout method for arranging items in rows or columns. It's perfect for navigation bars, card layouts, centering content, and distributing space between elements. CSS Grid is a two-dimensional layout system — it handles both rows and columns simultaneously.

Flexbox
One-dimensional (row OR column). Best for components: navbars, buttons, centering content.
Grid
Two-dimensional (rows AND columns). Best for page layouts, galleries, complex structures.

JavaScript — Behaviour

JavaScript is the programming language of the web. It runs directly in your browser and can respond to user actions, update the page content, fetch data from servers, and much more — all without reloading the page.

While HTML provides structure and CSS provides style, JavaScript provides behaviour — making pages interactive and dynamic.

How JavaScript Runs

JavaScript is an interpreted language — the browser reads and executes it line by line, top to bottom. It runs in the browser's built-in JavaScript engine (V8 in Chrome, SpiderMonkey in Firefox). No compilation step is needed.

adding-js.html 
<!-- 1. External file (recommended) — use defer so HTML loads first -->
<script defer src="script.js"></script>

<!-- 2. Inline script — JS written directly in the HTML file -->
<script>
  console.log("Page loaded!");
</script>

<!-- 3. Inline event handler — avoid this approach, hard to maintain -->
<button onclick="alert('clicked')">Click</button>
The console
console.log() prints values to the browser's DevTools console (F12). It's your best debugging tool — use it constantly while learning.
defer attribute
Tells the browser to download the script but wait until the HTML is fully parsed before running it. This ensures your JS can find all the HTML elements it needs.
Statements & semicolons
Each instruction is a statement. Statements end with a semicolon ;. JavaScript can often infer them (ASI), but writing them explicitly avoids subtle bugs.
Case sensitivity
JavaScript is case-sensitive. myVariable, MyVariable, and MYVARIABLE are three completely different things. By convention, use camelCase.
Variables
Used to store data such as text, numbers, or objects. Use const for values that won't change and let for values that can be updated.
Functions
Reusable blocks of code that perform a specific task. Functions help organize logic and avoid repeating code.

Data Types

Every value in JavaScript has a type. JavaScript is dynamically typed — you don't declare the type explicitly; the engine figures it out at runtime.

types.js 
// String — text, wrapped in quotes
const name    = "SkillLance";
const greeting = `Hello, ${name}!`;  // template literal — can embed expressions

// Number — integers and decimals (no separate int/float types)
const score   = 95;
const pi      = 3.14;

// Boolean — only two possible values
const passed  = true;
const failed  = false;

// Array — ordered list of values, zero-indexed
const colours = ["red", "green", "blue"];
console.log(colours[0]);  // "red"

// Object — key/value pairs, like a record
const user = {
  name:  "Alex",
  score: 95,
  passed: true
};
console.log(user.name);   // "Alex"

// null — intentionally empty value
// undefined — variable declared but not yet assigned
💡 typeof: Not sure what type a value is? Use typeof myValue — it returns a string like "string", "number", "boolean", or "object". Handy for debugging.

Variables, Functions & Data Types

Modern JavaScript uses const for values that don't change, and let for values that do. Avoid var — it has unintuitive scoping behaviour.

script.js 
// ── Variables ───────────────────────────────
const siteName = "SkillLance";     // string — immutable reference
let   count    = 0;              // number — can be reassigned
const isReady  = true;           // boolean

// ── Functions ───────────────────────────────
function greet(name) {
  return `Hello, ${name}!`;    // template literal
}

// Arrow function (ES6+) — concise syntax
const double = (n) => n * 2;

DOM Manipulation

The Document Object Model (DOM) is a tree-like representation of your HTML that JavaScript can read and modify. When you select an element and change its text or style with JavaScript, you are manipulating the DOM — and the browser updates the page instantly.

dom.js 
// ── Selecting elements ──────────────────────
const heading = document.getElementById("main-title");
const btn     = document.querySelector("button");

// ── Reading and writing content ─────────────
console.log(heading.textContent); // read
heading.textContent = "New Title";  // write — safer than innerHTML
heading.style.color = "#818cf8";   // change CSS

// ── Changing classes ─────────────────────────
heading.classList.add("highlight");     // add class
heading.classList.toggle("active");    // toggle on/off

Events

JavaScript responds to user interactions through events. An event listener is a function that runs when a specific event happens — a click, a keypress, scrolling, loading, and hundreds more.
